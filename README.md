# CMajor-FMOD-Wrapper

A wrapper that automatically compiles [Cmajor](https://cmajor.dev/) patches into FMOD DSP plugins (effects and instruments).

**Acknowledgments:** This project is based on the **Amorph** project by **Artists in DSP**. The included template is largely taken from it and adapted for this use as an FMOD plugin.

This tool integrates the Cmajor compiler directly into the CMake build process, automatically generating the necessary C++ code, FMOD parameter mappings, and multichannel expansion logic.

## Setup

1. Clone this repository.
2. Firelight Technologies does not allow the redistribution of their headers. Download the FMOD Engine API and copy the FMOD `api/core/inc` folder into the `CMake` folder so that the structure is `repo/CMake/inc/fmod.h`.

## Usage

There are two ways to compile a Cmajor patch into an FMOD plugin:

### Method 1: File-based

1. Place exactly one `.cmajor` file into the `SrcExportDir` directory.
2. Run CMake to configure and build:
   ```bash
   cd CMake
   cmake -B Build
   cmake --build Build
   ```
3. The compiled FMOD plugin (`.dylib`, `.dll`, `.so`, or `.bundle`) will be generated in the `BuildProducts` folder at the root of the repository.

*Note: The `.cmajorpatch` JSON file is automatically generated.*

### Method 2: Clipboard-based

If you have Cmajor code generated by an LLM or copied from an editor, you can compile it directly from your system clipboard without creating files manually.

1. Copy the raw Cmajor code to your clipboard.
2. Run CMake with the clipboard flag:
   ```bash
   cd CMake
   cmake -B Build -DUSE_CLIPBOARD=ON
   cmake --build Build
   ```
3. The script will automatically read your clipboard, detect the processor name, save the files to `SrcExportDir`, and build the plugin in `BuildProducts`.

## Features

- **Auto-Parameter Mapping**: `input event float` parameters are automatically exposed as FMOD parameters with their `min`, `max`, and `init` values.
- **External Buffer Data Parameters**: buffer endpoints can be exposed as FMOD `DATA` parameters for encoded audio uploads (WAV/MP3 via bundled decoders in `decoder/`).
- **Auto-Detection**: Automatically detects if the patch is an instrument (generator) or an effect based on the presence of `input stream` endpoints.
- **Multichannel Expansion**: Mono effects automatically expand to process stereo, 5.1, or 7.1 FMOD tracks by instantiating multiple Cmajor processors under the hood.
- **Auto-CLI Fetching**: Automatically downloads and caches the `cmaj` CLI tool if it isn't found on the system.

## Buffer Endpoint Contract

To expose an FMOD `DATA` parameter for uploading an encoded audio file:

1. Add this exact endpoint to your patch:
   `input event std::audio_data::Mono sampleBuffer [[ name: "Sample Buffer", fmodBuffer: true ]];`
2. Handle the event and store the content in state for use in `main()`.
3. The wrapper decodes dropped files and provides mono PCM to this endpoint.
